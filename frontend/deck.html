The Problem

ADAS sees the road.
Nothing watches inside.

❌ What ADAS does today

External perception — vehicles, pedestrians, lane markings.
Zero awareness of what's happening inside the cabin.

⚠ In-cabin anomalies go unmonitored

Rattling objects · crying children · pet distress · misused seatbelts · unsecured items before a pothole · driver fatigue from radar-detectable vitals.

🧠 The cognitive load gap

Each anomaly is minor alone. Together they create continuous micro-distractions that silently erode driver attention — especially at high speed or low visibility.

"Drivers experience continuous, unmonitored in-cabin anomalies acting as micro-distractions … without any intelligent system support."

— Problem Statement §2

The critical gap

No existing system fuses internal cabin context to actively detect → understand → mitigate distractions in real time.

Key Differentiator

mmWave radar as a
first-class cabin sensor

In-cabin mmWave radar (TI IWR6843 / AWR class) is already entering production vehicles. CabinSense treats it as a primary intelligence source, not a proximity sensor.

💓
Contactless driver vitals
Respiration rate + heart rate via micro-Doppler — a direct cognitive-load signal. No camera required.
🧑‍🤝‍🧑
Per-seat occupancy + occupant type
Adult / child / pet per seat, with motion energy. Corroborates audio events to eliminate false positives.
🔒
Privacy-preserving
Detects child presence and driver fatigue without storing video — critical for regulated markets.
🚨
Child-left-behind safety alert
Rear child + no driver detected → immediate escalation (horn / lights / notification). Zero false negatives.

Radar vitals in the demo

Every seat shows live HR and breathing rate. Under "driver stress" scenario: HR jumps to 104 bpm, resp. to 22 rpm — Cognitive Load score rises 30 points and the calming persona activates.

Production path

Swap sensors/radar.py with a UART parser for the IWR6843. Bus contract unchanged. Rest of the system is unaffected.

Three Use Cases · One Engine

What CabinSense does

🔊

1 — Audio comfort

Crying / pet distress detected by PyTorch AST model (live mic, 527 AudioSet classes) and corroborated by radar occupancy — no false alarm from the stereo. Lowers volume, warms AC, softens lighting.

🧘

2 — Driver persona tuning

Radar contactless vitals (HR + breathing) + camera face emotion → stress or fatigue detected → calming playlist, lighting colour, temperature adjustment. Confirm-first so driver stays in control.

radar vitalsvision
⚡

3 — Pre-emptive safety

IMU CSV replay (50 Hz) predicts rough road 4 s ahead. YOLO detects unsecured objects. Radar catches seatbelt misuse (hook-but-not-worn). Advisory fires before the pothole hits.

IMUvisionradar

💡 Idea #4 (ask before acting) is the UX principle across all three — every advisory is confirm-first. The system suggests; the driver decides.

Technical Architecture

How it's built

Sensing nodes (bus publishers)

📡
Radar
Synthetic mmWave — occupancy, HR, breathing, micro-motion per seat at 10 Hz
🎤
Audio — PyTorch AST
Browser mic → 16 kHz chunks → HuggingFace Audio Spectrogram Transformer (527 AudioSet classes)
📷
Vision — MediaPipe + YOLOv8n
Browser webcam → MediaPipe FaceLandmarker (EAR drowsiness) + YOLOv8n (objects at 5 fps)
📳
IMU / Vibration
50 Hz CSV replay → RMS road-quality classifier → pre-emptive pothole warning
🚗
Vehicle context
Speed, visibility, GPS-derived pothole distance

Fusion engine (10 Hz, <1 ms)

Subscribes to all bus topics. Computes the Cognitive-Load / Attention-Risk score (0–100) from weighted sensor signals. Selects and ranks mitigations. Preserves confirm/dismiss state across recomputes.

Delivery

FastAPI + WebSocket → 10 Hz fused state to the React Cabin HMI.
Latency measured at <1 ms (target: <200 ms).

Stack

Python 3.11 FastAPI PyTorch CPU MediaPipe 0.10 YOLOv8n React 18
