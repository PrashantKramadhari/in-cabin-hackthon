The Problem
Drivers Face Continuous, Unmonitored Micro-Distractions

Critical Gap

Modern vehicles monitor the road ahead with ADAS — but the cabin interior is completely unmonitored.

No infant / pet welfare

No object hazard detection

No cognitive load awareness

5 Real Scenarios

  • Baby crying while driving (distraction)
  • Child / pet left behind (safety)
  • Unsecured objects before a pothole (hazard)
  • Driver fatigue or stress (cognitive load)
  • Automatic cabin changes without consent (UX failure)

Fusion Engine · Core
Cognitive Load Score — Multi-Modal Weighted Accumulator
CL_score = 0
── Driver vitals (radar) ──────────────────
if HR > 95: CL += min(30, (HR−95)×1.5)
if RESP > 20: CL += min(15, (RESP−20)×2)
if HR < 60: CL += 12 # fatigue
── Driver emotion (vision / world) ────────
if stressed: CL += 10
if tired: CL += 8
── Audio (BabyNet + AST) ──────────────────
if crying|animal: CL += 25
if shouting: CL += 20
if rattle: CL += 10
── Child / pet (world.seats — no lag) ─────
per seat: CL += 5 + distress×40 + HR_over×1.5
── Road quality (IMU) ─────────────────────
if pothole: CL += 18  if rough: CL += 8
── Vision (Qwen + YOLO + MediaPipe) ───────
if drowsy: CL += 15
if loose_object: CL += 6–10
── Visibility / speed ─────────────────────
if vis=low: CL += 12  if spd>100: CL += 8
return min(100, CL)

Key Design Choices

  • Rule-based for full transparency (auditable)
  • world.seats read directly → zero radar lag on manual config
  • HR deduplication across radar + world seats
  • Additive accumulation → easy to extend with ML scorer
  • Capped at 100; each factor is independently traceable
  • Fusion runs at 10 Hz, measured latency <1 ms
Fusion Engine · Mitigations
Confirm-First · Non-Intrusive · Rule-Driven
Advisory
Soothe Cabin

Crying / animal sound + child/pet seat confirmed → lower media volume 30%, warm AC +1°C, soft lighting.

Calming Persona

Driver HR >95 or stressed emotion → calm playlist, cool blue lighting, temp −1°C.

Alertness Boost

Driver HR <60 or tired → upbeat playlist, bright lighting, fresh-air burst, break suggestion.

Unsecured Item on Seat

Qwen detects object on seat → advisory to stow before braking.

Cabin Monitoring Active

Child/pet detected, no audio anomaly → radar vitals monitoring confirmed, adjustable.

Warning
Seatbelt Not Engaged

Seat occupied (world.seats) but buckled=false → chime + visual, fires immediately on configure.

Secure Before Road Hazard

Loose object + pothole within 120 m → urgent secure advisory before impact.

Child / Pet Heartbeat Alert

Child HR >115 bpm (pet >125) → warning. >threshold+20 → escalates to critical.

Passenger Distress

Shouting detected → check passenger, consider pulling over.

Critical
CHILD PRESENCE ALERT

Child in rear, no driver present → horn + lights + owner notification. No confirm required.

⚡ Grace period 6s — confirmed/dismissed mitigations are retained for 6 seconds after the condition clears, preventing flicker from momentary sensor noise.
Sensing Stack
Four Modalities · Privacy-Preserving · Graceful Degrade

📡 Radar

mmWave
  • 4-seat occupancy map
  • Contactless HR (Doppler)
  • Respiration rate
  • Micro-motion (agitation)
  • Child-left-behind alert

Ref HW: TI IWR6843 / AWR class

🎙️ Audio

2-stage
  • BabyNet CNN — custom mel-spec, ~10ms GPU
  • Classes: none / crying / talking / happy
  • ≥0.60 conf → fast-path
  • AST fallback — MIT AudioSet 527-class
  • Detects: crying, barking, rattle, speech

📷 Vision

3-layer
  • Qwen2-VL-2B — per-seat: kind, emotion, objects, buckled
  • MediaPipe — EAR drowsiness, mouth emotion
  • YOLOv8n — loose object detection (COCO)
  • Priority: Qwen → MediaPipe → YOLO

📳 IMU + Vehicle

CAN / GPS
  • Road quality: smooth / rough / pothole
  • Pothole-ahead distance (m)
  • Speed (km/h)
  • Visibility: good / low / fog / rain
  • Pre-emptive object-securing trigger

Sensor Priority & Fallback

Radar occupancy → World state override → Qwen vision → MediaPipe  |  BabyNet fast-path → AST fallback  |  All nodes publish independently — fusion always has the latest frame per modality, never blocked.
AI Models
On-Device · No Cloud · Packaged in models/

Qwen2-VL-2B-Instruct

Qwen/Qwen2-VL-2B-Instruct

Vision-language model analyzing each camera frame. Returns structured JSON per seat: occupant kind, emotion, buckled state, and loose objects list.

4-bit quantized
~2.5 GB VRAM
100–150 ms/frame
RTX 3080

BabyNet CNN

models/baby_net.pt

Custom lightweight mel-spectrogram CNN trained on baby audio samples. Fast-path classifier with 4 classes. Falls back to AST on low confidence.

Custom trained
~458 KB
~10 ms GPU
4 classes

AST — AudioSet

MIT/ast-finetuned-audioset-10-10-0.4593

Audio Spectrogram Transformer fine-tuned on 527-class AudioSet. Fallback for BabyNet low-confidence frames. Detects: crying, barking, rattle, speech.

527 classes
~330 MB
CPU inference
Threshold 0.15

MediaPipe FaceLandmarker

models/face_landmarker.task

478 facial landmarks → EAR-based drowsiness detection and heuristic emotion (eye openness + mouth ratio). Runs as fallback when Qwen is absent.

float16
3.6 MB
Real-time
EAR thresh 0.20

YOLOv8n

models/yolov8n.pt

Nano YOLO for fast object detection in cabin frames. Identifies loose objects: backpack, suitcase, bottle, laptop, book. Triggers secure-object advisory.

COCO classes
6.3 MB
~10 ms
Conf threshold 0.35

Model Strategy

  • All models in models/ — fully offline
  • Qwen overrides MediaPipe/YOLO when available
  • BabyNet fast-path → AST fallback cascade
  • Vision override guard — manual seat config never overridden by Qwen
  • download_models.py for one-shot packaging

Technical Stack
Backend · Frontend · Comm · Models

Backend — Python 3.11

  • FastAPI + uvicorn — WebSocket + REST
  • bus.py — async pub/sub, bounded queue, <1ms
  • world.py — shared ground-truth cabin state
  • fusion.py — 10Hz fusion loop, rule-based scorer
  • sensors/ — 6 independent async nodes
  • scenarios.py — scripted demo mutations

Communication

  • WebSocket /ws — bidirectional JSON at 10Hz
  • Frontend → backend: audio_chunk, vision_frame, configure_seat, scenario, confirm/dismiss
  • Backend → frontend: full fused state per tick

Frontend — React 18 (zero-build)

  • CDN-free local libs (Babel 7.12.17 pinned)
  • SensorView: Video, Audio, Radar, Vehicle, IMU, Seat widgets
  • InfotainmentView: Circular gauge, Mitigation cards, Scenarios
  • TimelineBar: Mode indicator (Manual / Live A/V / Scenario)
  • RadarWidget: per-seat kind selector + HR sliders
  • Live WebSocket push — no polling

Key Invariants

  • Sensor nodes never block — each runs at its own Hz
  • Swap only the node body to go to real hardware
  • Fusion stays rule-based → ML scorer can replace _cognitive_load() behind same interface
  • HMI is build-free for hackathon speed
